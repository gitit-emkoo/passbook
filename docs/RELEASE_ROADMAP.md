### Phase 14: 빌드 및 배포 (1주)

#### 목표
- 프로덕션 빌드 생성
- 스토어 제출

#### 작업 내용

**1. 환경 변수 설정**
- [ ] 프로덕션 환경 변수 설정
  - API_URL (프로덕션 서버)
  - AdMob 실제 앱 ID
  - JWT_SECRET (강력한 비밀번호)
- [ ] 환경별 설정 분리

**2. 백엔드 배포**
- [ ] 프로덕션 서버 설정 (예: AWS, Heroku, Railway)
- [ ] 데이터베이스 마이그레이션
- [ ] 환경 변수 설정
- [ ] HTTPS 설정
- [ ] 도메인 연결

**3. 프론트엔드 빌드**
- [ ] EAS Build 설정 (`eas.json`)
- [ ] Android 프로덕션 빌드
  ```bash
  eas build --platform android --profile production
  ```
- [ ] iOS 프로덕션 빌드
  ```bash
  eas build --platform ios --profile production
  ```

**4. 스토어 제출**
- [ ] Google Play Console에 앱 업로드
- [ ] Apple App Store Connect에 앱 업로드
- [ ] 심사 제출

**5. 심사 대기**
- [ ] Google Play: 보통 1-3일
- [ ] Apple App Store: 보통 1-7일

## 결론

현재 Phase 1~8까지 기본 기능 구현이 완료되었으므로, 다음 단계로 **Phase 9: 테스트 및 버그 수정**부터 시작하는 것을 추천합니다.
